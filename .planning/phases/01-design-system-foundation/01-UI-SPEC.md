## Dark Mode Implementation

**Strategy:** Class-based toggle via Alpine.js

**Implementation:**
```javascript
// In app layout or global Alpine component
Alpine.store('darkMode', {
    enabled: localStorage.getItem('darkMode') !== 'false',
    toggle() {
        this.enabled = !this.enabled;
        localStorage.setItem('darkMode', this.enabled);
        document.documentElement.classList.toggle('light', !this.enabled);
    }
})
```

**CSS Variables:** Already defined in `resources/css/app.css`
- Dark mode: `:root` (default)
- Light mode: `.light` class

**Theme Toggle Component:** New component for Phase 1
- Sun/moon icon toggle
- Positioned in navigation header
- Persists preference in localStorage
- Respects system preference on first visit

## Accessibility Requirements

| Requirement | Implementation |
|-------------|----------------|
| Color contrast | WCAG AA minimum (4.5:1 for text, 3:1 for UI components) |
| Focus indicators | `outline-2 outline-offset-2 outline-primary` on all interactive elements |
| Keyboard navigation | All interactive elements reachable via Tab key |
| Screen reader support | Semantic HTML, ARIA labels for custom components |
| Touch targets | Minimum 44px × 44px on mobile devices |
| Motion preferences | Respect `prefers-reduced-motion` for animations |

**Animation Strategy:** Subtle transitions (150-200ms) for hover states, fade-in animations for page loads. No jarring movements.
