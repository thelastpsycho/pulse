# CSS Component Classes Defined in app.css
- .btn: Base button class with focus states and transitions
- .btn-primary: Pink-500 gradient for primary actions
- .btn-secondary: Surface-2 background with border
- .btn-danger: Red-500 for destructive actions
- .btn-success: Green-500 for confirm actions (already defined in app.css)
- .badge: Base badge class with rounded-full and uppercase text
- .badge-success, .badge-warning, .badge-danger, .badge-muted: Color variants
- .card: Card container with gradient background and border

# Component Requirements from UI-SPEC.md
- Primary buttons: Use pink-500 gradient (from-indigo-500 to-pink-500)
- Badges: Status pills with color coding (Open=muted, In Progress=warning, Closed=success)
- Cards: Consistent container with gradient backgrounds and borders
- All components: Support dark mode via CSS variables, focus states, keyboard navigation
