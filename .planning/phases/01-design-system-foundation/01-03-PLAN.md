# Responsive Breakpoints (from UI-SPEC.md)
- Mobile First: 0px - 640px (phones)
- sm: 640px (phones landscape, small tablets)
- md: 768px (tablets portrait)
- lg: 1024px (tablets landscape, small desktops)
- xl: 1280px (desktops)
- 2xl: 1536px (large desktops)

# Mobile-First Strategy
- Base styles target mobile (no breakpoint prefix)
- md: and lg: modifiers for larger screens
- Touch targets: minimum 44px × 44px on mobile
- No horizontal scrolling at any breakpoint

<task type="auto">
  <name>Task 2: Fix responsive navigation layout</name>
  <files>resources/views/layouts/navigation.blade.php, resources/views/layouts/app.blade.php</files>
  <read_first>resources/views/layouts/navigation.blade.php, resources/views/layouts/app.blade.php</read_first>
  <action>
    Ensure the navigation layout works properly on mobile, tablet, and desktop.

    Check navigation.blade.php for:
    1. Mobile menu trigger (hamburger button) visible only on small screens
    2. Desktop navigation visible only on large screens
    3. Proper Alpine.js state for mobile menu open/close
    4. Theme toggle accessible on all screen sizes
    5. User menu/dropdown works on touch devices

    Typical responsive navigation pattern:
    ```blade
    <!-- Mobile menu button -->
    <button x-data="{ open: false }" @click="open = !open" class="lg:hidden">
        <!-- Hamburger icon -->
    </button>

    <!-- Mobile menu (hidden on desktop) -->
    <div x-show="open" class="lg:hidden">
        <!-- Mobile navigation links -->
    </div>

    <!-- Desktop navigation (hidden on mobile) -->
    <nav class="hidden lg:flex">
        <!-- Desktop navigation links -->
    </nav>
    ```

    Requirements:
    - Mobile menu should use lg:hidden (hidden on desktop, visible on mobile)
    - Desktop nav should use hidden lg:flex (visible on desktop, hidden on mobile)
    - Mobile menu items should be full-width and have adequate spacing
    - Touch targets minimum 44px height on mobile
    - Dropdowns should work with click (not hover) on touch devices
    - Theme toggle should be visible in both mobile and desktop nav

    Check app.blade.php for:
    - Proper viewport meta tag: <meta name="viewport" content="width=device-width, initial-scale=1">
    - Container max-width with responsive classes: max-w-7xl mx-auto px-4 sm:px-6 lg:px-8
  </action>
  <verify>
    <automated>grep -n "lg:hidden\|hidden lg:" resources/views/layouts/navigation.blade.php | wc -l | xargs -I {} test {} -gt 0 exits 0</automated>
  </verify>
  <done>Navigation has responsive breakpoints with mobile menu trigger and desktop layout. Mobile menu is accessible via hamburger button. Theme toggle visible on all screen sizes. Touch targets are adequate on mobile.</done>
</task>
