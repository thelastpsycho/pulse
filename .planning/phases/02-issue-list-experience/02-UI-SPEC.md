## Visual Hierarchy

### Primary Screen: Issue List Page (Enhanced)

**Focal Point Declaration:**

1. **Primary (First eye draw):** Page heading "Issues" with View Toggle (left) + Create Issue button (right, pink-500 gradient)
2. **Secondary:** Filter row (search input + dropdown filters + Save Filter button)
3. **Tertiary:** Saved filter chips (horizontal scrollable list below filters)
4. **Quaternary:** Issue list/table OR Kanban board (depending on view toggle)
5. **Quinary:** Bulk action toolbar (slides down when issues selected)
6. **Footer:** Pagination controls

**Visual Flow (Table View):**
```
Page Title (H1) + View Toggle → [Create Issue] → Filters → Saved Filters → Issue Table → Bulk Toolbar → Pagination
```

**Visual Flow (Kanban View):**
```
Page Title (H1) + View Toggle → [Create Issue] → Filters → Saved Filters → Kanban Columns (Open | In Progress | Closed)
```

**Layout Structure:**
- **Top bar:** Page heading (left), View Toggle (center-left), Create Issue button (right)
- **Below heading:** Filter row (search input + status dropdown + priority dropdown + department dropdown + Save Filter button)
- **Below filters:** Saved filter chips (horizontal scroll, gap-2)
- **Main content:**
  - **Table view:** Scrollable table with checkbox column, status badges, inline action buttons
  - **Kanban view:** Three columns (Open, In Progress, Closed) with draggable cards
- **Bottom (conditional):** Bulk action toolbar (fixed position, slides down from top)
- **Footer:** Pagination controls (centered)

**Purpose:** User's primary goal is efficiently viewing, filtering, and managing issues. View toggle provides flexibility for different workflows. Saved filters reduce repetitive filter setup. Bulk actions speed up multi-issue operations.

## Interaction Patterns

### Kanban Board Drag-and-Drop

**Library:** SortableJS (integrate via npm)

**Behavior:**
- **Drag start:** Card lifts with 4px translateY, 50% opacity, shadow-xl
- **Drag over:** Column shows 2px dashed indigo-500 border at drop position
- **Drop:** Card snaps to new position, Livewire updates issue status
- **Drop animation:** 200ms fade-in + slide-in animation
- **Touch support:** 44px minimum touch target on drag handle

**Visual States:**
- **Default:** Card with border, hover effect
- **Dragging:** 50% opacity, scaled 1.02, shadow-2xl
- **Drop zone:** Column background shifts to surface-2, 2px dashed border
- **After drop:** 300ms success flash (green glow on updated card)

**Accessibility:**
- Keyboard: Tab to focus card, Space to enter "drag mode", Arrow keys to move between columns, Enter to drop
- Screen reader: "Draggable issue card, currently in Open column. Press Space to drag."
- Focus indicators: 2px indigo-500 outline on focused card

---

### Bulk Selection

**Behavior:**
- **Individual checkbox:** Toggle selection state, update bulk toolbar count
- **Select all:** Checkbox in table header toggles all visible issues
- **Bulk toolbar:** Appears when 1+ issues selected, shows count + action buttons
- **Deselect:** Click outside selected issues or X in toolbar

**Visual States:**
- **Unselected:** Default checkbox style
- **Selected:** Indigo-500 background, white checkmark, 4px solid border, 8px inset shadow
- **Partial selection (some selected):** Indeterminate checkbox state (dash icon)

**Bulk Action Buttons:**
- **Close:** Success button (green gradient)
- **Assign:** Secondary button with indigo-500 border
- **Change Status:** Secondary button with indigo-500 border
- **Cancel (clear selection):** Ghost button (text only)

---

### Quick View Modal

**Behavior:**
- **Trigger:** Click "View Details" button or issue title (table view) / card (Kanban view)
- **Open:** 200ms fade-in + scale-in animation from center
- **Close:** Click X button, press Escape, click backdrop (with 150ms fade-out)
- **Content:** Read-only issue details (no editing in quick view)

**Layout:**
- **Header:** Issue ID + Title (left) + Close button (right)
- **Body:**
  - Status badge + Priority badge + Department badge
  - Description (with markdown rendering)
  - Created by + Created at
  - Assigned to + Due date
  - Recent activity (last 3 comments)
- **Footer:** "View Full Details" button (navigates to issue detail page)

**Visual States:**
- **Backdrop:** 50% opaque black overlay
- **Modal:** Surface background, 2xl rounded corners, shadow-2xl
- **Max width:** 600px (desktop), 100% (mobile with margin-4)

---

### Saved Filters

**Behavior:**
- **Save:** Click "Save Filter" button → Modal appears → Enter name → Save
- **Apply:** Click saved filter chip → Filters update, list refreshes
- **Delete:** Right-click chip → "Delete" option (or long-press on mobile)
- **Edit:** Click chip → Load filter → Modify → Save as new or overwrite

**Visual States:**
- **Default:** Surface-2 background, border, muted text
- **Active (applied):** Primary (indigo-500) border, primary foreground text, glow effect
- **Hover:** Border shifts to primary/50, cursor pointer
- **Overflow:** Horizontal scroll for more than 5 chips

## Dark Mode Implementation

**Strategy:** Class-based toggle via Alpine.js (inherited from Phase 1)

**Implementation:**
```javascript
// In app layout or global Alpine component
Alpine.store('darkMode', {
    enabled: localStorage.getItem('darkMode') !== 'false',
    toggle() {
        this.enabled = !this.enabled;
        localStorage.setItem('darkMode', this.enabled);
        document.documentElement.classList.toggle('light', !this.enabled);
    }
})
```

**CSS Variables:** Already defined in `resources/css/app.css`
- Dark mode: `:root` (default)
- Light mode: `.light` class

**Theme Toggle Component:** Existing from Phase 1
- Sun/moon icon toggle
- Positioned in navigation header
- Persists preference in localStorage
- Respects system preference on first visit

**Phase 2 Dark Mode Considerations:**
- **Kanban columns:** Ensure column background differences are visible in both modes (use opacity shifts)
- **Drag ghost:** Test opacity in both light and dark modes
- **Drop zone indicator:** Ensure indigo-500 border is visible on both backgrounds
- **Bulk selection:** Primary (indigo-500) must work on both surface colors

## Accessibility Requirements

| Requirement | Implementation |
|-------------|----------------|
| Color contrast | WCAG AA minimum (4.5:1 for text, 3:1 for UI components) |
| Focus indicators | `outline-2 outline-offset-2 outline-primary` on all interactive elements |
| Keyboard navigation | All interactive elements reachable via Tab key, including Kanban cards |
| Screen reader support | Semantic HTML, ARIA labels for custom components (drag-and-drop, modals) |
| Touch targets | Minimum 44px × 44px on mobile devices |
| Motion preferences | Respect `prefers-reduced-motion` for animations (disable drag animations) |
| Drag-and-drop keyboard | Alternative to mouse dragging: Space to enter drag mode, Arrow keys to move, Enter to drop |

**Animation Strategy:** Subtle transitions (150-200ms) for hover states, fade-in animations for page loads. No jarring movements.

**Phase 2 Accessibility Additions:**
- **Kanban drag-and-drop:**
  - ARIA drag attributes: `draggable="true"`, `aria-grabbed`, `aria-dropeffect`
  - Keyboard: Space to grab, Arrow keys to move, Enter to drop, Escape to cancel
  - Screen reader announcements: "Moved to In Progress column"
- **Bulk selection:**
  - ARIA: `aria-selected` on selected rows/cards
  - Live region: Announce selection count changes
- **Quick view modal:**
  - ARIA: `role="dialog"`, `aria-modal="true"`, `aria-labelledby`
  - Focus trap: Keep focus within modal when open
  - Escape key: Close modal
- **View toggle:**
  - ARIA: `role="tablist"`, `aria-selected`, `aria-controls`
  - Keyboard: Arrow keys to switch between tabs
